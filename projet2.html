Collecte automatisée de données web


Le web est une mine de données fascinantes qui peuvent enrichir considérablement un entrepôt de données. Dans le cadre de mon projet, je vais me concentrer sur la mise en place de méthodes pour alimenter un entrepôt de données avec des données provenant du web. Que ce soit en entreprise ou au sein d'une organisation, je vais devoir utiliser différentes méthodes pour récupérer ces données, que ce soit via des API de fournisseurs de données ou en développant mes propres solutions d'intégration et de qualification des données. En tant qu'étudiant en informatique décisionnelle, je vais également devoir réfléchir à la structuration de l'entrepôt de données pour qu'il soit prêt à accueillir ces données web, ainsi qu'à l'architecture logicielle nécessaire pour garantir la collecte et l'intégrité de ces données.

Les objectifs de ce projet sont les suivants :

Comprendre la spécificité des données web.
Découvrir le rôle crucial des interfaces de programmation d'application (API) dans la récupération de données web.
Approfondir ma réflexion sur les processus de structuration et d'alimentation d'un entrepôt de données avec des données provenant du web.

Description

Dans le cadre de ce projet, je vais être amené à collecter et à intégrer des données provenant du web dans un entrepôt de données. Pour répondre à un besoin d'analyse à des fins décisionnelles, je vais devoir mobiliser l'ensemble des compétences acquises au cours du semestre. Cela impliquera la conception et la mise en œuvre d'un entrepôt de données capable d'accueillir des données provenant de sources en ligne. Après avoir établi la structure de l'entrepôt, je devrai développer un programme pour récupérer les données sur le web (par exemple, en utilisant des API) et les traiter (nettoyage, pré-traitement, jointures...) avant de les stocker dans l'entrepôt de données.

Contraintes :

- Difficultés d'élaboratin du code (fonctions)
- Adaptation du code selon la source
